Demo scenarios

What I demonstrated

FIM + VirusTotal + Active Response

When a suspicious file appears in a monitored folder, Wazuh enriches the alert with threat intelligence, raises severity, sends notification, and can trigger a controlled endpoint response.

  • Mapped file creation to malware reputation context.
  • Connected alert severity to response policy.
  • Documented email/dashboard evidence for analyst workflow.

Security Configuration Assessment and vulnerability context

The lab intentionally used weak password policy and outdated software to demonstrate how SOC visibility can include posture management, not only attack detection.

  • Tracked failed CIS-style checks and remediation guidance.
  • Mapped software inventory to CVE exposure.
  • Showed how risk can be prioritized before exploitation.

Web attack alert to firewall block

ModSecurity generated web attack logs, Wazuh correlated the alert, and the response path demonstrated how a web-layer event can lead to a network-layer containment decision.

  • Correlated WAF logs with Wazuh rules.
  • Triggered controlled pfSense blocking.
  • Reduced detection-to-containment time in lab conditions.

Active Directory brute-force detection

Repeated failed authentication generated Windows Security events. Wazuh correlation mapped the activity to credential access behavior and triggered endpoint containment in the lab.

  • Used Event ID 4625 as identity telemetry.
  • Mapped detection to brute-force technique context.
  • Validated network isolation through endpoint firewall state.
