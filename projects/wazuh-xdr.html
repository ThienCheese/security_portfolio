How each demo works

The diagrams below show the full path from event source to detection, response, and analyst evidence.

1. Malware C2 detection and endpoint response

Source

Suspicious file appears

Endpoint folder is monitored by Wazuh FIM.

Collect

FIM alert

File hash and path become investigation context.

Enrich

VirusTotal lookup

External reputation converts file change into malware signal.

Analyze

High-severity rule

Rule logic marks the event as response-worthy.

Respond

Controlled removal

Active Response removes the malicious file in the lab.

Report

Email + dashboard

Analyst receives alert evidence and audit trail.

2. SCA and CVE posture management

Baseline

Weak policy / old app

Lab intentionally introduces risky configuration and software.

Scan

SCA + inventory

Wazuh scans security configuration and installed packages.

Match

CIS + CVE context

Failed checks and vulnerable software are mapped to risk.

Prioritize

Risk view

Findings help fix exposure before active exploitation.

Remediate

Hardening path

Dashboard gives evidence and remediation direction.

3. Web attack alert to firewall block

Attack

SQLi-style request

Attacker traffic reaches the DMZ web application.

Detect

ModSecurity log

WAF identifies abnormal web input and writes a log.

Correlate

Wazuh rule

Web alert is converted into actionable security signal.

Trigger

Active Response

Rule ID is tied to a controlled firewall action.

Contain

pfSense block

Firewall prevents repeat access from the attacker source.

Validate

Rule evidence

Dashboard and firewall state prove the response path.

4. AD brute-force detection and endpoint isolation

Behavior

Repeated failed logons

Authentication failures target an AD account in the lab.

Telemetry

Windows Event 4625

Domain Controller provides identity security evidence.

Correlation

Brute-force rule

Wazuh groups failures and maps the behavior to credential access.

Response

Endpoint firewall policy

Active Response changes firewall state on the suspected machine.

Contain

Inbound/outbound blocked

The machine is isolated to limit continued password guessing.

Evidence

Ping failure + alert

Network test and dashboard confirm containment.

What I want to demonstrate

This project is not only a tool installation. It shows the SOC workflow: collect telemetry, normalize it, map it to rule context, decide whether response is safe, and preserve evidence for investigation.

Production considerations

For real environments, response rules need whitelist handling, rollback, alert tuning, change control, and incident lifecycle integration before automatic containment is enabled.
